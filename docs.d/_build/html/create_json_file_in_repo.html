create_json_file_in_repo module¶

When clonezilla images are created we should upload the information to artifactory so it is in a format that the tbsinstaller can use.

create_json_file_in_repo.download_file(url_to_download, api_key, save_file_as)¶

Download some information from artifactory.

create_json_file_in_repo.get_data_from_rest(rest_interface, username, password, keyvalue)¶

Query a rest interface to get a key. In the usage for clonezilla is to get tbsclonezillas api key

create_json_file_in_repo.get_hardware_from_mdb(hostname)¶

Try and determine the hardware used by a machine based on information we have in mdb which itself is gathered from dmidecode.

create_json_file_in_repo.modify_names(dictionary_name, new_image_name, hostname, tbs_release_type)¶

Depending on the hardware or release type, modify the dictionary, the keys and the json file name.

create_json_file_in_repo.post_to_artifactory(url_to_upload_to, username, api_key, data)¶

Put our json/dictionary into artifactory

create_json_file_in_repo.read_current_json(json_file)¶

If we already have some json in artifactory read that in as a python dictionary

create_json_file_in_repo.run_a_command(cmd_to_run)¶

Run a standard linux command on a system.

create_json_file_in_repo.upload_json_to_artifactory()¶

This function pulls in all the other functions to upload the correctly formatted json to artifactory.

create_json_file_in_repo.write_to_json(dictionary_name)¶

Temporarily write the json file to a system so we can upload
